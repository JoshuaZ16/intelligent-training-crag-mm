## 第 1 周个人周报

一、组号

20

二、实训题目

面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现

三、本周小组分工

本周小组围绕 CRAG-MM 赛题开展项目启动工作。小组两名成员采用模块分工方式：一名成员主要负责模型生成、Prompt 设计、Agent 策略和后续多轮上下文处理；另一名成员主要负责环境配置、检索 API 调试、数据集加载、本地评测和错误案例整理。两人共同阅读项目文档，讨论单源增强、多源增强和多轮问答三个任务的递进关系，并制定四周实训计划。

四、本人负责的部分

本人本周主要负责赛题理解、任务拆解和初步 Agent 策略设计。具体包括阅读 CRAG-MM starter kit 的 README、agent 开发说明、数据集文档和检索 API 文档，梳理单源增强、多源增强、多轮问答三个任务的输入输出、检索源差异和评分要求。同时，我负责设计第一版回答生成 Prompt，明确系统需要基于图片和检索证据作答，在信息不足时主动回答 `I don't know`，避免无依据生成。

五、针对本人负责的部分，所做的工作及完成情况，反思等

本周我完成了三个任务的技术路线梳理，并根据课程要求制定了四周递进式计划：先跑通单源图像检索增强，再加入网页检索完成多源融合，最后扩展到多轮上下文问答。在分析过程中，我认识到该赛题的关键不只是让模型回答问题，更重要的是控制幻觉、筛选证据并保证答案简洁。当前不足是对检索 API 返回结果的利用方式还不够成熟，后续需要结合实际样例继续优化 Prompt 和证据组织方式。

## 第 2 周个人周报

一、组号

20

二、实训题目

面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现

三、本周小组分工

本周小组继续围绕任务1和任务2开展实训。两名成员保持模块分工：模型生成与 Agent 策略负责人主要负责任务1 Prompt 优化、任务2多源融合 Prompt 设计和中期汇报中的方法部分；检索与评测负责人主要负责网页检索 API 接入、检索结果字段整理、小规模评测和错误案例分析。两人共同完成任务1优化、任务2原型设计和中期线下汇报准备。

四、本人负责的部分

本人本周主要负责任务1的回答策略优化和任务2的多源融合方案设计。在任务1中，我重点调整 Prompt，使模型能够区分图片可见内容和图像检索得到的结构化知识，减少直接猜测。在任务2中，我设计了将图片摘要、用户问题、图像 KG 信息和网页检索结果共同组织为证据上下文的方案，并参与准备中期线下汇报中的方法设计部分。

五、针对本人负责的部分，所做的工作及完成情况，反思等

本周我完成了任务1 Prompt 的初步优化，并参与构建任务2的多源增强流程。通过分析样例，我发现模型错误主要来自两类情况：一是检索结果中存在无关信息，二是模型在证据不足时仍倾向于生成看似合理的答案。因此我在 Prompt 中加入了证据优先和不确定拒答的约束。中期汇报准备过程中，我进一步明确了项目的递进逻辑，即从单源检索到多源融合再到多轮对话。后续需要继续提升网页检索结果的筛选质量。

## 第 3 周个人周报

一、组号

20

二、实训题目

面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现

三、本周小组分工

本周小组重点推进任务2优化和任务3原型。模型生成与 Agent 策略负责人主要负责多源证据融合 Prompt、多轮上下文 Prompt、问题类型划分和回答生成策略优化；检索与评测负责人主要负责任务2不同版本的对比实验、典型错误案例整理、任务3数据集接入和多轮评测流程测试。两人共同分析任务2中的检索噪声和任务3中的上下文依赖问题。

四、本人负责的部分

本人本周主要负责任务2生成策略优化和任务3多轮上下文问答方案设计。在任务2中，我围绕多源证据融合继续调整 Prompt，使模型优先使用与问题最相关的检索片段，并尽量输出简洁答案。在任务3中，我负责分析 `message_histories` 的格式，设计如何在后续轮次中结合历史问题、历史回答和当前图片信息，保持回答的一致性。

五、针对本人负责的部分，所做的工作及完成情况，反思等

本周我完成了任务3多轮问答的初步设计，并对任务2的多源融合策略进行了反复调整。通过对错误样例的分析，我发现多轮任务的难点在于后续问题常常省略实体名称，需要模型从历史对话中恢复指代；同时，如果前面回答错误，后续回答也容易受到影响。因此我设计了更明确的上下文组织方式，让模型在回答当前问题时同时参考历史对话和当前问题需求。本周的不足是任务3还需要更多样例验证，下一步将重点减少上下文冲突和连续错误。
