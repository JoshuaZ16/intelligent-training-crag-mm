# 课程实训材料索引

本目录用于存放智能技术实训的项目计划、周报、线下汇报和最终报告结构。

## 文件说明

- `4-week-execution-plan.md`  
  4 周整体执行计划，包括项目定位、小组分工、每周目标、两次线下汇报安排和最终提交材料清单。

- `weekly-reports-personal.md`  
  按“模型生成与 Prompt/Agent 策略负责人”视角写好的 4 份个人周报成稿，可直接作为本人周报基础。

- `teammate-weekly-report-guide.md`  
  按“检索、评测与实验负责人”视角写好的队友周报参考，方便两人保持前三部分一致、第四和第五部分突出差异。

- `presentation-and-report-outline.md`  
  第 2 周中期汇报、第 4 周总结汇报和最终 6-8 页实训报告的结构提纲。

- `week1-2-runbook.md`  
  重做后的环境、Agent 模式、固定样例、真实实验与合成契约验收命令。

- `week1-2-rework-acceptance.md`  
  按“真实验证 / 合成验证 / 待云端验证”区分的前两周正式验收状态。

- `week2-error-analysis.md`  
  已修复工程问题、合成案例、真实数据观察和云端实验记录模板。

## 正式交付位置

- `deliverables/week1-2-weekly-reports/`：张家豪两份周报与俞凡两份待本人确认周报。
- `deliverables/中期汇报/CRAG-MM-第20组-中期汇报草稿.pptx`：7 页可编辑中期汇报草稿。
- `artifacts/week2/`：环境快照、固定样例、原始 trace、合成汇总、真实模型冒烟记录与文档/PPT 渲染证据。

## 推荐使用顺序

1. 先阅读 `4-week-execution-plan.md`，确定每周目标和小组分工。
2. 每周提交周报时，从 `weekly-reports-personal.md` 复制对应周内容，再按实际完成情况微调。
3. 队友可参考 `teammate-weekly-report-guide.md` 填写个人贡献部分。
4. 第 2 周和第 4 周汇报前，按 `presentation-and-report-outline.md` 制作 PPT。
5. 第 4 周撰写实训报告时，按 `presentation-and-report-outline.md` 的报告结构展开。
