# 汇报与实训报告结构

## 1. 第 2 周中期线下汇报 PPT

建议页数：6-8 页。

建议时长：5-8 分钟。

### 第 1 页：题目与小组信息

标题：

**面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现**

内容：

- 组号：20
- 小组成员：2 人
- 课程：智能技术实训
- 汇报类型：中期汇报

### 第 2 页：赛题背景

要点：

- CRAG-MM 是面向多模态、多轮问答的 RAG benchmark。
- 应用场景包括智能眼镜、可穿戴设备和视觉问答。
- 核心挑战是减少幻觉，提高事实性和上下文一致性。

### 第 3 页：三项任务递进关系

要点：

- 任务1：单源增强，只使用图像检索和结构化 KG。
- 任务2：多源增强，加入网页检索。
- 任务3：多轮问答，加入历史对话上下文。
- 项目采用“单源 -> 多源 -> 多轮”的递进路线。

### 第 4 页：任务1已完成内容

要点：

- 已阅读项目文档并跑通 starter kit。
- 已梳理 Agent 接口和本地评测流程。
- 已设计任务1原型流程：图片 -> 图像检索 -> KG 信息筛选 -> Prompt -> 答案生成。
- 当前主要问题：无关 KG 字段干扰、模型可能猜测、答案需要更简洁。

### 第 5 页：任务2初步方案

要点：

- 在任务1基础上加入 web search。
- 使用图片摘要和用户问题构造检索查询。
- 将图像 KG 和网页 snippet 分开组织为证据。
- 通过 Prompt 要求模型优先依据证据，不确定时回答 `I don't know`。

### 第 6 页：当前问题与后续计划

要点：

- 问题：检索噪声、幻觉、速度限制、75 token 截断、多轮上下文依赖。
- 第 3 周：优化任务2，建立错误分析表，启动任务3。
- 第 4 周：完善任务3，整理实验结果，完成报告和最终 PPT。

## 2. 第 4 周总结汇报 PPT

建议页数：10-14 页。

建议时长：10 分钟汇报 + 5 分钟问答。

### 第 1 页：题目与小组信息

标题：

**面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现**

内容：

- 组号：20
- 小组成员：2 人
- 课程：智能技术实训
- 汇报类型：总结汇报

### 第 2 页：项目背景与问题

要点：

- VLLM 在视觉问答中容易出现事实幻觉。
- RAG 通过外部知识检索增强模型回答。
- CRAG-MM 同时考察多模态理解、检索增强和多轮对话。

### 第 3 页：任务定义

要点：

- 任务1：单源图像 KG 增强。
- 任务2：图像 KG + 网页检索多源增强。
- 任务3：多轮上下文一致问答。

### 第 4 页：系统整体架构

建议画流程：

```text
输入图片 + 用户问题
        |
图像理解 / 图片摘要
        |
图像检索 KG ---------
        |             |
网页检索 / 查询改写   |
        |             |
证据筛选与融合 <------
        |
Prompt 构造
        |
答案生成
        |
答案后处理 / 拒答判断
```

### 第 5 页：任务1方法

要点：

- 使用图像检索 API 获取相似图片和结构化实体属性。
- 从 KG 字段中筛选与问题相关的信息。
- Prompt 明确区分视觉信息和检索证据。

### 第 6 页：任务2方法

要点：

- 使用图片摘要 + 用户问题构造网页检索查询。
- 图像 KG 负责识别实体，网页检索补充开放事实。
- 将两类证据分块输入模型，降低信息混杂。

### 第 7 页：任务3方法

要点：

- 使用 `message_histories` 引入历史对话。
- 根据当前问题判断是否依赖历史实体、图片或新检索。
- 在 Prompt 中要求回答保持上下文一致，避免与前文冲突。

### 第 8 页：核心改进与创新点

要点：

- 多源证据组织：图像 KG 与网页 snippet 分离。
- 查询改写：结合图片摘要和用户问题提升检索命中率。
- 拒答策略：证据不足时回答 `I don't know`，降低幻觉。
- 多轮上下文管理：利用历史对话恢复指代并保持一致性。

### 第 9 页：实验设置

要点：

- 数据集：CRAG-MM single-turn 和 multi-turn validation split。
- baseline：RandomAgent、LlamaVisionModel 或 SimpleRAGAgent。
- 指标：accuracy、missing rate、hallucination rate、truthfulness score、多轮 conversation score。
- 说明官方限制：离线环境、10 秒 x batch size、75 BPE token 截断。

### 第 10 页：实验结果

建议表格：

| 版本 | 任务 | 准确率 | Missing | Hallucination | Truthfulness |
| --- | --- | --- | --- | --- | --- |
| Baseline | Single-turn | 待填 | 待填 | 待填 | 待填 |
| Task1 Agent | Single-source | 待填 | 待填 | 待填 | 待填 |
| Task2 Agent | Multi-source | 待填 | 待填 | 待填 | 待填 |
| Task3 Agent | Multi-turn | 待填 | 待填 | 待填 | 待填 |

### 第 11 页：案例分析

建议展示 2-3 个案例：

- 成功案例：检索证据命中，答案简洁正确。
- 失败案例：检索结果噪声导致答案偏离。
- 多轮案例：后续问题通过历史上下文恢复指代。

### 第 12 页：总结与反思

要点：

- 完成了从单源到多源再到多轮的完整系统。
- 多模态 RAG 的关键在于检索质量、证据组织和生成约束。
- 幻觉控制比单纯增加信息源更重要。
- 后续可加入重排序模型、置信度估计和更强视觉语言模型。
