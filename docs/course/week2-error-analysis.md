# 第二周错误分析与迭代记录

## 1. 已发现并修复的工程问题

| 问题 | 影响 | 修复与证据 |
| --- | --- | --- |
| 旧 Agent 不区分任务模式 | 任务1仍尝试网页检索 | 新增 `vision/task1/task2` 显式模式；集成测试证明 task1 网页调用为 0 |
| KG 直接取前若干字段 | 无关字段挤占上下文 | 将字段解析为结构化证据，按问题词与字段/实体重合度排序并去重 |
| 检索异常被静默吞掉 | 无法区分无结果和系统故障 | 记录异常类型、状态和拒答原因到 JSONL trace |
| 重复图片摘要推理 | 增加超时风险 | Task2 优先使用 image KG 实体构造查询，避免额外摘要模型调用 |
| 回答数量可能与批次不一致 | 结果错位 | 增加输入长度和后端返回数量校验，并覆盖失败测试 |
| 旧测试锁定旧入口 | 新版本无法成为默认 Agent | 更新入口断言，同时保留旧原型辅助测试 |
| 独立环境缺少 `loguru` | 实验脚本无法启动 | 加入 `requirements-macos.txt` 并通过 `--help` 启动检查 |

## 2. 合成契约案例

合成案例只用于验证行为，不表示真实效果。

- Task1 成功路径：问题询问建筑师，KG 包含 `architect` 字段，30/30 正确提取；Vision 对同一契约样例安全拒答。
- Task2 网页增益路径：KG 只有产品类别，网页摘要提供价格，Task1 30/30 拒答，Task2 30/30 从网页证据回答。
- 噪声过滤：空摘要、重复 URL+摘要和低于阈值的网页结果会在进入 Prompt 前删除。
- 异常路径：图像检索抛出异常时，Agent 返回规范拒答，trace 标记 `search_error`，而不是伪装为正常无结果。

## 3. 真实数据观察

已下载 CRAG-MM single-turn validation 的最后一个 parquet 分片并抽取 30 条固定样例：

- 5 种 `query_category`，数量分别为 7、7、6、5、5。
- 5 个领域，数量分别为 7、7、6、5、5。
- 样例包含地点来源、汽车设计师奖项、食物发酵影响、植物果实颜色等，需要视觉识别与外部知识结合的问题。

目前尚未形成真实 Task1/Task2回答和评分，原因不是代码测试失败，而是完整检索索引规模与当前下载带宽不满足当日执行窗口。不得在中期材料中把本节写成真实性能对比。

本地 MLX 冒烟测试已使用真实 validation 图片完成。SmolVLM-500M-4bit 在 3.113 秒内输出 `Chicken curry`，而图片和数据描述对应 Kadhi Chawal。该案例标记为“视觉实体识别错误”，说明轻量模型适合验证接口与时延，不适合作为真实性指标基线；原始记录见 `artifacts/week2/real_model_smoke/smolvlm-500m.json`。

## 5. 下一轮迭代规则

1. Task1真实性得分未高于 Vision Baseline：先检查 image KG 命中率，再调整字段排序和拒答条件。
2. Task2低于 Task1：提高网页阈值、减少网页条数；仍退化时启用“KG 不足才使用网页”的门控。
3. P95 总耗时超过 10 秒：减小 batch、证据长度和检索结果数，不增加第二次模型推理。
4. 幻觉率升高：优先收紧证据要求，不以降低 Missing 为唯一目标。
