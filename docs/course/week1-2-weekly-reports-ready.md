二、实训题目

三、本周小组分工

本周小组继续围绕任务1和任务2开展实训。成员 A 主要负责任务1 Prompt 优化、任务2多源融合 Prompt 设计和中期汇报中的方法设计部分；成员 B 主要负责网页检索 API 接入、检索结果字段整理、小规模评测和错误案例分析。两人共同完成任务1优化、任务2原型设计和中期线下汇报准备。

四、本人负责的部分

本人本周主要负责任务1的回答策略优化和任务2的多源融合方案设计。在任务1中，我重点调整 Prompt，使模型能够区分图片可见信息和图像检索得到的结构化知识，减少直接猜测。在任务2中，我设计了将图片摘要、用户问题、图像 KG 信息和网页检索结果共同组织为证据上下文的方案，并将图像证据与网页证据分块输入模型，避免不同来源的信息混杂。

五、针对本人负责的部分，所做的工作及完成情况，反思等

本周我完成了任务1 Prompt 的初步优化，并参与构建任务2的多源增强流程。代码层面已将默认随机 Agent 切换为课程第二周版本 Agent，支持图像 KG 证据、网页 snippet 证据、查询改写、短答案和证据不足拒答约束。通过分析样例，我发现模型错误主要来自两类情况：一是检索结果中存在无关信息，二是模型在证据不足时仍倾向于生成看似合理的答案。因此我在 Prompt 中加入了证据优先和不确定拒答的约束。后续需要继续提升网页检索结果的筛选质量，并增加更多任务2样例评测。
