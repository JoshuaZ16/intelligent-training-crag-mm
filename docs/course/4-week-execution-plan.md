# 智能技术实训 4 周执行计划

## 1. 项目定位

实训题目建议统一为：

**面向 CRAG-MM 的多模态多轮检索增强问答系统设计与实现**

项目按 CRAG-MM 的三个任务递进推进：

1. **任务1：单源增强**  
   使用图像检索 mock API，从图片关联的结构化知识图谱中获取信息，完成基础多模态 RAG 问答。

2. **任务2：多源增强**  
   在任务1基础上加入网页检索 mock API，将图像 KG 信息与网页片段融合，提升事实覆盖范围和答案可靠性。

3. **任务3：多轮问答**  
   在多源增强能力基础上加入历史对话建模，解决连续追问、指代恢复和上下文一致性问题。

最终汇报叙事不把某一个任务孤立展示，而是突出：**从单源多模态 RAG 到多源检索增强，再到多轮上下文一致问答的完整系统**。该叙事最贴合课程评分中的“解决方案完整性”和“解决方案新颖性”。

## 4. 两次线下汇报安排

### 第 2 周中期汇报

建议结构：

1. 赛题背景：CRAG-MM 面向多模态 RAG 和智能眼镜问答。
2. 三任务递进关系：单源 -> 多源 -> 多轮。
3. 任务1已完成内容：图像检索、KG 信息组织、初步回答生成。
4. 任务2初步方案：网页检索、多源证据融合。
5. 当前问题：检索噪声、模型幻觉、答案截断、速度限制。
6. 后两周计划：任务2优化、任务3上下文建模、最终报告和 PPT。

### 第 4 周总结汇报

建议结构：

1. 项目背景：多模态 RAG 和 CRAG-MM 任务。
2. 任务递进：任务1解决图像 KG 单源增强，任务2解决图像 + 网页多源融合，任务3解决多轮上下文问答。
3. 系统架构：图像理解、检索、证据融合、答案生成、后处理。
4. 核心创新：查询改写、多源证据组织、拒答策略、多轮上下文状态管理。
5. 实验结果：baseline 对比、三个任务的表现、典型案例。
6. 反思展望：检索噪声、模型幻觉、时间限制、未来可加入重排序模型或更强 VLM。
